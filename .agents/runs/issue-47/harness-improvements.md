# 하네스 건강도 평가

## 총점

80/100

## 요약

#47은 대시보드 데이터 스키마와 다음 작업 후보 표시를 개선해 실행 기록의 가시성을 높였다. 다만 사용자 결정이 필요한 상태에서 `spec.md`가 먼저 작성됐다가 수정된 흐름은 하네스 역할 인수인계 규칙의 약점으로 남았다.

## 강점

- 활성 실행과 보관 실행 구분, 데이터 품질 경고가 대시보드 데이터에 들어갔다.
- GitHub 열린 이슈와 inbox 후보가 다음 작업 데이터로 노출됐다.
- `validate-dashboard-data.mjs`가 대시보드 데이터 계약을 검증한다.

## 약점

- spec 작성 전에 사용자 결정 확인을 강제하는 규칙이 충분히 반복되어 있지 않았다.
- Playwright 스냅샷 실패를 보완하는 DOM 수준 브라우저 검증은 추가되지 않았다.
- 기존 대시보드 문구 인코딩 깨짐은 별도 정리가 필요하다.
- 평가 산출물 일부가 영어 설명 문장으로 작성되어, 한국어 하네스 문서 흐름과 맞지 않았다.

## 개선 적용

- `2026-05-14-pre-spec-decision-gate`: 미확정 결정이 있으면 `spec.md`를 만들지 않는 규칙을 `ai-harness-spec`과 이슈 실행 워크플로에 추가했다.
- `2026-05-14-harness-korean-output-default`: 하네스 리뷰와 건강도 평가 산출물은 기본 한국어로 작성한다는 규칙을 리뷰/평가 스킬, 프롬프트, 워크플로, 인코딩 가이드에 반복해서 추가했다.

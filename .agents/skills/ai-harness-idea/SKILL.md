# AI 하네스 아이디어

가장 처음 실행하는 단계다. 다음에 무엇을 할지 사용자와 함께 정한다.

## 목적

- 현재 프로젝트 상태를 확인한다.
- 다음에 할 수 있는 작업 후보를 만든다.
- 각 후보의 목적, 가치, 리스크, 크기를 정리한다.
- 사용자와 함께 하나의 큰 아이디어를 선택한다.

## 절차

1. 현재 상태 확인
   - 필요하면 `ai-harness-status` 관점으로 진행 중인 작업과 누락 단계를 확인한다.
   - GitHub Issue를 만들거나 파일을 수정하지 않는다.

2. 후보 만들기
   - 다음 작업 후보를 2-3개만 제안한다.
   - 각 후보에는 목표, 기대 효과, 대략적 범위, 리스크를 포함한다.

3. 사용자 선택
   - 사용자가 선택하거나 수정할 때까지 기다린다.
   - 선택이 모호하면 한 가지 질문만 한다.

4. 다음 단계 안내
   - 선택된 큰 아이디어는 `ai-harness-plan`으로 넘긴다.
   - 이 단계에서는 이슈를 생성하지 않는다.

## 출력 형식

```md
다음 작업 후보

1. 후보명
- 목표:
- 기대 효과:
- 대략적 범위:
- 리스크:

추천:

질문:
```

## 금지

- GitHub Issue를 생성하지 않는다.
- `task-spec.md`나 `plan.md`를 작성하지 않는다.
- 코드를 수정하지 않는다.
- 검증이나 리뷰를 수행하지 않는다.

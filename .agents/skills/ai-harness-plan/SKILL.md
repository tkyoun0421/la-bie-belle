# AI 하네스 계획

선택된 큰 아이디어를 세부 실행 계획과 이슈 초안으로 나눈다.

## 목적

- 큰 아이디어를 실행 가능한 작업 단위로 쪼갠다.
- 각 작업의 목표, 범위, 제외 범위, 완료 기준, 검증 방법, 리스크를 정리한다.
- GitHub Issue로 만들 초안을 작성한다.
- 사용자 승인을 받기 전에는 이슈를 생성하지 않는다.

## 절차

1. 아이디어 확인
   - `ai-harness-idea`에서 선택된 큰 아이디어를 확인한다.
   - 저장소에서 확인 가능한 구조와 기존 맥락을 직접 확인한다.

2. 작업 분해
   - 큰 아이디어를 1개 이상의 실행 가능한 작업으로 나눈다.
   - 너무 큰 작업은 여러 이슈 후보로 분리한다.

3. 이슈 초안 작성
   - 각 이슈 후보에 대해 제목, 본문, 라벨을 작성한다.
   - 본문에는 목표, 범위, 제외 범위, 관련 파일/영역, 검증 방법, 완료 기준, 리스크를 포함한다.

4. 사용자 승인 대기
   - 이슈 초안을 사용자에게 보여준다.
   - 사용자가 승인하기 전에는 GitHub Issue를 생성하지 않는다.

5. 승인 후 안내
   - 승인되면 GitHub Issue 생성 단계로 넘어갈 수 있다.
   - 이슈가 생성된 뒤에는 해당 이슈 번호로 실행 기록을 만들고 Red 단계로 진행한다.

## 출력 형식

```md
계획안

큰 아이디어:

작업 분해:

이슈 초안:
- 제목:
- 라벨:
- 본문:

확인 필요:
이 초안으로 GitHub Issue를 생성할까요?
```

## 금지

- 사용자 승인 없이 GitHub Issue를 생성하지 않는다.
- 코드를 수정하지 않는다.
- Red/Green 구현을 시작하지 않는다.
- 검증이나 리뷰를 수행하지 않는다.

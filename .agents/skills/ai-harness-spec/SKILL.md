# AI 하네스 스펙

생성된 이슈 하나를 구현 가능한 상세 스펙으로 구체화한다.

## 선행 조건

다음 파일이 없으면 진행하지 말고 누락 파일을 보고한다.

- `.agents/runs/issue-{number}/task-spec.md`
- `.agents/runs/issue-{number}/plan.md`

## 절차

1. 원본 GitHub Issue, `task-spec.md`, `plan.md`를 읽는다.
2. 구현 전 확정해야 할 세부 동작을 정리한다.
3. 정상 흐름과 엣지케이스를 구분한다.
4. 에러, 빈 상태, 로딩 상태, 권한, 입력 검증, 데이터 규칙을 필요한 만큼 정의한다.
5. 테스트 케이스를 작성하고 Red 단계에서 먼저 작성할 실패 테스트 우선순위를 정한다.
6. `spec.md`만 작성하거나 갱신한다.

## 산출물

- `.agents/runs/issue-{number}/spec.md`

## 금지

- 코드를 수정하지 않는다.
- 테스트를 작성하지 않는다.
- Green 구현을 시작하지 않는다.
- 검증이나 리뷰를 수행하지 않는다.

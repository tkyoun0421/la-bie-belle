# Implementer 프롬프트

현재 GitHub Issue에 대해 승인된 계획을 구현한다.

입력:

- 원본 GitHub Issue
- `task-spec.md`
- `plan.md`
- 관련 저장소 파일

필수 산출물:

- `implementation-notes.md`: 변경 파일, 중요한 결정, 계획에서 벗어난 지점, 알려진 리스크.

규칙:

- 변경 범위를 작업 명세 안으로 제한한다.
- 기존 저장소 패턴을 따른다.
- 관련 없는 코드를 다시 작성하지 않는다.
- 구현 중 계획이 틀렸다는 점이 드러나면 멈추고 차단 사유를 기록한다.

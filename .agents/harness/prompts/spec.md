# Spec 프롬프트

생성된 GitHub Issue를 구현 가능한 상세 스펙으로 구체화한다.

입력:

- 원본 GitHub Issue
- `task-spec.md`
- `plan.md`
- 관련 저장소 파일

필수 산출물:

- `spec.md`: 사용자 시나리오, 정상 흐름, 엣지케이스, 에러/빈/로딩 상태, 권한, 입력 검증, 데이터 규칙, UI 동작, API 또는 모듈 계약, 테스트 케이스, Red 단계 우선순위.

규칙:

- 코드를 수정하지 않는다.
- 테스트를 작성하지 않는다.
- 구현 전에 모호한 동작을 가능한 한 명확히 한다.
- Red 단계가 바로 실패 테스트를 작성할 수 있도록 테스트 케이스를 구체화한다.

# Verifier 프롬프트

구현 결과를 작업 명세와 완료 기준에 맞춰 검증한다.

입력:

- 원본 GitHub Issue
- `task-spec.md`
- `plan.md`
- `implementation-notes.md`
- 변경 파일

필수 산출물:

- `verification.md`: 실행한 명령, 결과, 수동 점검, 완료 기준 매핑, 건너뛴 검증, 남은 리스크.

규칙:

- 테스트, 린트, 타입체크, 빌드, 집중 수동 점검에서 나온 직접 근거를 우선한다.
- 정확한 명령과 결과를 기록한다.
- 근거 없이 완료 기준을 검증됨으로 표시하지 않는다.
- 검증을 실행할 수 없다면 이유와 예상 리스크를 기록한다.

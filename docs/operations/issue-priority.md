# Issue Priority

Issue numbers are creation order, not execution order.

Priority labels:

- `priority:p0`: blocking foundation work
- `priority:p1`: core MVP work
- `priority:p2`: quality, usability, and operational improvements
- `priority:p3`: later cleanup or ideas

Status label:

- `status:blocked`: blocked by dependency or external decision

New issues may start with `needs-triage`. After triage, keep exactly one `priority:*` label.

Choose next work by excluding `status:blocked`, then selecting `priority:p0`, `priority:p1`, `priority:p2`, and `priority:p3` in order.
